## 📌 文档维护规范

### 命名规范
- 格式：`{编号}-{分类}-{文档名称}.md`
- 编号：两位数字，按分类范围编号
- 分类：需求与规划、架构设计、技术流程、事件中心、其他
- 文档名称：简洁明了，使用中文

### 分类编号范围
- 01-09: 需求与规划
- 10-19: 架构设计
- 20-29: 技术流程
- 30-39: 事件中心
- 40-49: （预留）前端开发
- 50-59: （预留）测试相关
- 60-69: （预留）运维部署
- 90-99: 其他

### 新增文档流程
1. 确定文档分类
2. 在对应分类范围内选择编号
3. 按照命名规范创建文档
4. 更新本索引文档
5. 记录更新日志
